# Prior-Art Research: The Digital Core Methodology

## Synthesis: Where PSDC Is Novel

The Digital Core Methodology describes a **configuration layer** (rules, skills, hooks, chronicle, memory) sitting atop Claude Code that transforms an LLM from plausible fabricator into verified research partner. This is genuinely novel in several ways:

1. **Configuration as epistemology, not just convenience.** Unlike frameworks that manage agents at the harness level (AutoGen, LangGraph, Cursor), PSDC operates *on top* of a fixed harness via behavioral rules + persistent state files. This makes verification and continuity human-readable and versionable.

2. **Hand-authored structured memory (chronicle + memory files) over learned compression.** Memory systems like Mem0 and MemGPT use neural summarization; PSDC uses dated markdown files written by the human-AI dyad, designed for re-findability and narrative coherence, not token efficiency.

3. **Verification-first research collaboration.** Research automation systems (AI Scientist, GPT Researcher, STORM) run end-to-end autonomously. PSDC keeps the human in the loop with explicit verification checkpoints and evidence documentation, operating closer to Engelbart's vision of augmentation than pure automation.

4. **Tools-for-thought extended to human-AI pairs.** Zettelkasten, Obsidian, Roam, and Bret Victor's work all extend human cognition through external structures. PSDC extends that lineage to human-AI cognition, where the dyad is the cognitive unit, not the individual.

PSDC thus sits at the intersection of: agent harnesses (Claude Code), memory architectures (Chronicle + MEMORY.md), research automation (but human-steered), and philosophical lineage (Engelbart + Extended Mind). It is neither fully autonomous nor fully manual—it is augmentative and verifiable by design.

## 1. Agent Harnesses & Frameworks

Agent harnesses are software systems that give LLMs access to tools (code execution, file systems, APIs) and coordinate their actions across multiple steps. PSDC operates *on top* of Claude Code; it does not build the harness itself.

### Claude Code (Anthropic)
- **What it is:** Anthropic's official CLI agent harness + IDE integration that reads codebases, plans tasks, executes via real development tools (editors, terminals, git), and evaluates results.
- **Layer:** Foundational harness—provides the execution environment.
- **How PSDC differs:** Claude Code is the underlying harness. PSDC is the *behavioral and state configuration* layer on top of it. Claude Code handles tool execution; PSDC adds rules (semantic logging, breath checkpoints), persistent memory (chronicle), and verification workflows.
- **Ref:** [Claude Code overview](https://code.claude.com/docs/en/overview); [Claude Agent SDK](https://www.anthropic.com/engineering/building-agents-with-the-claude-agent-sdk)

### Cursor (IDE-Integrated Agent)
- **What it is:** VS Code fork with native AI agent integration. Agents plan and execute code changes, run tests, and iterate. Supports parallel agent execution on isolated git worktrees.
- **Layer:** IDE layer—agent lives inside editor.
- **How PSDC differs:** Cursor treats the IDE as the operating environment. PSDC treats the terminal + file system as the operating environment. Cursor's strength is visual feedback for coding; PSDC's is persistent narrative state across multi-turn research.
- **Ref:** [Cursor Features](https://cursor.com/features); [Cursor Agent](https://cursor.com/agents)

### AutoGen (Microsoft)
- **What it is:** Framework for multi-agent conversation orchestration. Agents can be configured with different roles, tools, and LLMs, then orchestrated via chat-like interaction patterns.
- **Layer:** Orchestration layer—coordinates multiple agents.
- **How PSDC differs:** AutoGen orchestrates agent-to-agent conversations programmatically. PSDC maintains human-agent conversation state via persistent files (chronicle, memory). AutoGen is fully autonomous; PSDC is human-steered.
- **Ref:** [AutoGen GitHub](https://github.com/microsoft/autogen); [AutoGen v0.4](https://www.microsoft.com/en-us/research/articles/autogen-v0-4-reimagining-the-foundation-of-agentic-ai-for-scale-extensibility-and-robustness/)

### LangGraph (LangChain)
- **What it is:** Low-level orchestration framework for stateful agents using graph structures (nodes = functions, edges = state transitions). Provides durable execution, memory, and human-in-the-loop capabilities.
- **Layer:** State management + orchestration.
- **How PSDC differs:** LangGraph manages state in a graph database. PSDC uses human-readable markdown files (chronicle). LangGraph's state is opaque to the human; PSDC's is fully inspectable and version-controllable via git.
- **Ref:** [LangGraph GitHub](https://github.com/langchain-ai/langgraph); [LangGraph Docs](https://docs.langchain.com/oss/python/langgraph/overview)

### SWE-Agent (Princeton + Stanford, NeurIPS 2024)
- **What it is:** Agent-computer interface for autonomous software engineering. Takes a GitHub issue and automatically generates fixes using custom tool abstractions optimized for code generation.
- **Layer:** Task-specific harness (code tasks).
- **How PSDC differs:** SWE-Agent is designed for end-to-end autonomous task completion on a specific domain (GitHub issues). PSDC is domain-agnostic and research-focused; it emphasizes verification and narrative continuity over autonomous completion.
- **Ref:** [SWE-agent GitHub](https://github.com/SWE-agent/SWE-agent); [ArXiv](https://arxiv.org/abs/2405.15793)

### OpenHands / OpenDevin (All-Hands-AI)
- **What it is:** Open-source AI software engineer. Runs autonomous agents in a sandboxed compute environment with access to shell, code editor, and browser.
- **Layer:** Autonomous execution environment.
- **How PSDC differs:** OpenHands executes tasks autonomously end-to-end. PSDC keeps the human in the loop with explicit checkpoints (breath, verification gates). OpenHands excels at task completion; PSDC excels at understanding and proof.
- **Ref:** [OpenHands GitHub](https://github.com/OpenHands/OpenHands); [OpenHands Docs](https://openhands.dev/)

### Devin (Cognition AI)
- **What it is:** Commercial AI software engineer agent. Handles complex multi-step engineering tasks with planning, memory, and self-correction. Evaluated on SWE-bench.
- **Layer:** Autonomous execution (commercial).
- **How PSDC differs:** Devin is a closed-source commercial system optimized for engineering velocity. PSDC is open-source configuration + behavior. Devin automates; PSDC augments.
- **Ref:** [Devin AI](https://devin.ai/); [Cognition Blog](https://cognition.ai/blog/introducing-devin)

### CrewAI
- **What it is:** Framework for orchestrating role-based agents that collaborate via shared context and delegation. Includes memory management (short-term, long-term, entity, contextual).
- **Layer:** Multi-agent orchestration + memory.
- **How PSDC differs:** CrewAI treats agents as autonomous collaborators with managed memory. PSDC treats the human-agent dyad as the primary cognitive unit. CrewAI's memory is learned compression; PSDC's is hand-authored narrative.
- **Ref:** [CrewAI GitHub](https://github.com/crewaiinc/crewai); [CrewAI Docs](https://docs.crewai.com/en/introduction)

## 2. LLM Memory & Continuity Systems

Memory systems enable agents to persist information across sessions, recall relevant context, and learn from past interactions.

### Letta (formerly MemGPT)
- **What it is:** Platform for building stateful agents with advanced memory. Uses a hierarchical memory system inspired by OS-level virtual memory: core memory (persistent context) and archival memory (out-of-context database, retrieved on demand).
- **How it relates:** Letta's core/archival distinction is mechanically similar to PSDC's approach (immediate context via chronicle, long-term via MEMORY.md), but Letta automatically compresses memories via neural summarization. PSDC uses hand-authored markdown—more interpretable, less token-efficient.
- **Key difference:** Letta optimizes for token efficiency; PSDC optimizes for narrative coherence and re-findability.
- **Ref:** [Letta GitHub](https://github.com/letta-ai/letta); [MemGPT Paper](https://arxiv.org/abs/2310.08560)

### Mem0
- **What it is:** Universal memory layer for LLM applications. Extracts, stores, and retrieves relevant facts from interactions. Supports episodic, semantic, procedural, and associative memory types.
- **How it relates:** Mem0 automatically manages memory lifecycle (extraction → storage → retrieval). PSDC's human writes chronicle entries manually, then PSDC scans them for externalization to MEMORY.md. Mem0 is automated; PSDC is deliberate.
- **Key difference:** Mem0 reduces token cost by ~90%; PSDC trades token efficiency for human interpretability.
- **Ref:** [Mem0 GitHub](https://github.com/mem0ai/mem0); [Mem0 ArXiv](https://arxiv.org/abs/2504.19413)

### Generative Agents (Stanford, Park et al., 2023)
- **What it is:** Agents with memory architecture supporting observation → reflection → planning cycles. Agents store experiences as natural language, synthesize them into higher-level reflections, and retrieve them dynamically.
- **How it relates:** Core/reflection/retrieval mirrors PSDC's chronicle (observation) → memory externalization (reflection) → consultation pattern. Both treat memory as interpretable natural language, not vectors.
- **Key difference:** Generative Agents were designed for believable social simulation. PSDC is designed for research verification.
- **Ref:** [Generative Agents Paper](https://arxiv.org/abs/2304.03442); [ACM UIST](https://dl.acm.org/doi/abs/10.1145/3586183.3606763)

### CoALA: Cognitive Architectures for Language Agents (Princeton, 2023)
- **What it is:** Conceptual framework organizing agents along three dimensions: memory (working + long-term), action space (internal + external), decision-making (planning + execution).
- **How it relates:** CoALA is a *taxonomy* of agent memory patterns. PSDC instantiates one specific point in that space: hand-authored working memory (chronicle) + persistent long-term memory (MEMORY.md) + deliberate planning checkpoints (breath).
- **Key difference:** CoALA describes the design space; PSDC is a specific realization optimized for verification-first research.
- **Ref:** [CoALA ArXiv](https://arxiv.org/abs/2309.02427); [CoALA GitHub](https://github.com/ysymyth/awesome-language-agents)

### Voyager (NVIDIA, 2023)
- **What it is:** Lifelong learning agent in Minecraft that explores autonomously, acquires skills, and builds a reusable skill library from executed code.
- **How it relates:** Voyager's skill library (interpretable, reusable code) parallels PSDC's skills directory. Both treat executable patterns as primary knowledge artifacts.
- **Key difference:** Voyager learns skills autonomously via trial and error. PSDC's skills are human-designed or human-approved behavioral configurations.
- **Ref:** [Voyager Paper](https://arxiv.org/abs/2305.16291); [NVIDIA Blog](https://blogs.nvidia.com/blog/ai-jim-fan/)

## 3. AI-Assisted Research Pipelines

Research automation systems aim to accelerate or fully automate components of the scientific discovery process.

### The AI Scientist (Sakana AI, 2024 + v2, 2025)
- **What it is:** Fully autonomous system for scientific research. Generates novel research ideas, implements them in code, runs experiments, visualizes results, and writes papers. AI Scientist-v2 uses agentic tree search for generalization across ML subfields.
- **How PSDC differs:** AI Scientist runs end-to-end autonomously with no human intervention. PSDC keeps the human in the loop at verification gates. AI Scientist costs ~$15/paper; PSDC invests human attention to ensure claims are grounded. AI Scientist is maximally autonomous; PSDC is maximally verifiable.
- **Notable:** AI Scientist-v2 produced the first workshop paper written entirely by AI and accepted through peer review (2025).
- **Ref:** [AI Scientist](https://sakana.ai/ai-scientist/); [AI Scientist v2 GitHub](https://github.com/SakanaAI/AI-Scientist-v2); [Nature Article](https://www.nature.com/articles/d41586-026-00899-w)

### GPT Researcher
- **What it is:** Autonomous agent for deep online research. Plans research questions, iteratively searches multiple sources, extracts and validates content, then synthesizes into a detailed report (5-6 pages).
- **How PSDC differs:** GPT Researcher automates web research and synthesis. PSDC coordinates verification *and* web research, using the same principles (multiple sources, cross-reference, evidence trails). GPT Researcher generates reports; PSDC generates verified knowledge artifacts (papers, datasets, proofs).
- **Ref:** [GPT Researcher GitHub](https://github.com/assafelovic/gpt-researcher)

### Stanford STORM (2024)
- **What it is:** System for synthesizing topic outlines through retrieval and multi-perspective question asking. Simulates conversations between writers with different perspectives and a topic expert, generates comprehensive Wikipedia-like articles.
- **How PSDC differs:** STORM automates the pre-writing research phase. PSDC coordinates the entire research workflow from problem definition through verification. STORM is perspective-rich; PSDC is verification-first.
- **Ref:** [STORM GitHub](https://github.com/stanford-oval/storm); [STORM Paper](https://arxiv.org/pdf/2402.14207)

### AutoSurvey (automated literature review)
- **What it is:** Systems that automatically generate literature reviews by querying multiple databases, synthesizing papers, and organizing findings.
- **How PSDC differs:** AutoSurvey automates survey generation. PSDC includes literature review as one component within a larger verification and narrative structure. Neither is novel in isolation; the novelty is the integration.
- **Ref:** Various implementations; conceptually similar to STORM + GPT Researcher combination.

## 4. Philosophical & Cognitive Lineage

PSDC inherits from 60+ years of thought about how humans extend cognition via external structures and tools.

### Licklider: "Man-Computer Symbiosis" (1960)
- **Core idea:** Humans should cooperate with machines in a symbiotic partnership where machines handle "routinizable work" and humans set goals, formulate hypotheses, evaluate criteria.
- **How PSDC instantiates it:** PSDC is a concrete instantiation of Licklider's vision. The human (Wisdom) sets research direction; Claude (PSDC) handles tool execution, memory management, and evidence gathering. The rules and checkpoints (breath, verification) formalize Licklider's evaluation loop.
- **Ref:** [Licklider 1960](https://groups.csail.mit.edu/medg/people/psz/Licklider.html); [PDF](https://worrydream.com/refs/Licklider_1960_-_Man-Computer-Symbiosis.pdf)

### Engelbart: "Augmenting Human Intellect" (1962)
- **Core idea:** Computing should augment human capability for problem-solving via cooperative human-machine systems, not replace it. The H-LAM/T framework (Human, tools, Language, Artifacts, Methodology).
- **How PSDC instantiates it:** Rules, skills, and chronicle are Engelbart's "artifacts" and "methodology" layer. The human-agent dyad forms the "H-LAM/T" system. Verification checkpoints (breath) formalize reflection on the augmentation process itself.
- **Ref:** [Augmenting Human Intellect](https://dougengelbart.org/content/view/138/); [Archive](https://archive.org/details/1962-engelbart-AHI-framework)

### Clark & Chalmers: "The Extended Mind" (1998)
- **Core idea:** Cognitive processes "ain't all in the head." External objects (notebooks, instruments, software) can be constitutive parts of the cognitive system, not just aids.
- **How PSDC instantiates it:** The chronicle, MEMORY.md, and skills files are constitutive of PSDC's cognition. They are not auxiliary—they are part of the cognitive system. The human-agent dyad with these structures is the extended mind.
- **Ref:** [Clark & Chalmers 1998](https://academic.oup.com/analysis/article-abstract/58/1/7/153111); [MIT Press](https://mitpress.mit.edu/9780262014038/the-extended-mind/)

### Clark: "Supersizing the Mind" (2008)
- **Core idea:** Human cognition extends through embodied, situated, distributed cognition. Thought is "inextricably tied to bodily form, habits of action and intervention, and the enabling web of social, cultural, and technological scaffolding."
- **How PSDC instantiates it:** The scaffold (rules, skills, chronicle, hooks) is the technological scaffolding Clark describes. The human-agent system is embedded in that scaffold.
- **Ref:** [Supersizing the Mind](https://global.oup.com/academic/product/supersizing-the-mind-9780195333213); [Oxford](https://academic.oup.com/book/6746)

### Hutchins: "Cognition in the Wild" (1995)
- **Core idea:** Cognition is distributed across people, artifacts, and environments—not confined to individual heads. Navigation, piloting, and problem-solving are inherently socially and materially distributed.
- **How PSDC instantiates it:** Research cognition in PSDC is distributed across Wisdom (decision-maker), Claude (executor), and the file system (memory/artifact store). None of these alone constitutes the cognitive system.
- **Ref:** [Cognition in the Wild](https://mitpress.mit.edu/9780262581462/cognition-in-the-wild/); [UCSD](https://pages.ucsd.edu/~ehutchins/citw.html)

### Zettelkasten & Luhmann (1950s–present)
- **Core idea:** Knowledge management via networked notes. Niklas Luhmann's Zettelkasten (90,000+ notes in his lifetime) demonstrated that personal knowledge systems based on connection, not just collection, enable extraordinary productivity.
- **How PSDC relates:** Chronicle + MEMORY.md + cross-referencing mirrors Zettelkasten philosophy. PSDC treats dated files and explicit links as primary knowledge artifacts, not secondary summaries.
- **Ref:** [Zettelkasten Method](https://zettelkasten.de/introduction/); [Luhmann](https://www.sloww.co/zettelkasten/)

### Obsidian, Roam, and Graph-Based PKM (2020s)
- **Core idea:** Personal knowledge management via graph-based linking and visualization. Roam and Obsidian make Zettelkasten principles digital and networked.
- **How PSDC relates:** PSDC does not use graph UI but does use the same principles (linked markdown files, dated organization, cross-referencing). The difference is that PSDC links are explicit and writable by the human-agent dyad.
- **Ref:** [Obsidian](https://obsidian.md/); [Roam Research](https://roamresearch.com/); [Comparison](https://deepterm.app/blog/obsidian-vs-roam-research-knowledge-management-comparison)

### Bret Victor: "Tools for Thought" (2010s–present)
- **Core idea:** Computing media should be *thinking tools* that provide immediate feedback, dynamic visualization, and explorable explanations. Bad representations hinder understanding; good ones unlock it.
- **How PSDC relates:** Chronicle entries and memory files are thinking tools in Victor's sense. They provide feedback loops (read-write-reflect-revise). The goal is not efficiency but understanding.
- **Ref:** [Inventing on Principle](https://worrydream.com/ABriefRantOnTheFutureOfInteractionDesign/); [Dynamicland](https://www.media.mit.edu/events/bret-victor-talk/)

### Heinz von Foerster: Second-Order Cybernetics (1970s)
- **Core idea:** In first-order cybernetics, the observer is separate from the observed. Second-order cybernetics acknowledges the observer as part of the system being observed. Knowledge is constructed by the observer.
- **How PSDC relates:** PSDC formalizes this via rules that make observation explicit (semantic logging, breath checkpoints). The human observer is not detached; their thinking and verification are part of the system.
- **Ref:** [Second-Order Cybernetics](https://en.wikipedia.org/wiki/Second-order_cybernetics); [Von Foerster](https://cepa.info/fulltexts/2758.pdf)

### Donella Meadows: "Thinking in Systems" (1997/2008)
- **Core idea:** Systems have leverage points—places where small changes produce large effects. The deepest leverage points are paradigms and mental models.
- **How PSDC relates:** Behavioral rules (semantic logging, breath) are leverage points in the system. They reshape how the human-agent dyad thinks about research (verification-first, narrative-continuous).
- **Ref:** [Leverage Points Essay](https://donellameadows.org/archives/leverage-points-places-to-intervene-in-a-system/); [Thinking in Systems](https://research.fit.edu/media/site-specific/researchfitedu/coast-climate-adaptation-library/climate-communications/psychology-amp-behavior/Meadows-2008.-Thinking-in-Systems.pdf)

### Andrej Karpathy: "Software 2.0 / 3.0" (2017–2025)
- **Core idea:** Programming paradigm shift from explicit code (Software 1.0) to data + neural networks (Software 2.0) to natural language prompts (Software 3.0).
- **How PSDC relates:** PSDC sits between Software 2.0 and 3.0. It treats rules and skills as the primary substrate (closer to 1.0's explicitness) but accepts that LLM reasoning is the engine. The configuration layer is the "code."
- **Ref:** [Software 2.0](https://karpathy.medium.com/software-2-0-a64152b37c35); [Latent Space on Software 3.0](https://www.latent.space/p/s3)

### Pattie Maes: Computational Reflection (1987)
- **Core idea:** Systems that can reason about their own operation. Reflection enables a system to inspect and modify its own behavior.
- **How PSDC relates:** Rules and hooks enable computational reflection. The human-agent dyad can inspect its own behavior (via chronicle) and modify it (via rule updates).
- **Ref:** [Maes 1987](https://dl.acm.org/doi/10.1145/38807.38821); [PDF](https://www.laputan.org/pub/foote/Reflection.pdf)

### Christopher Alexander: "A Pattern Language" (1977)
- **Core idea:** Complex systems (buildings, towns, communities) can be designed via patterns—recurring solutions to recurring problems. Patterns form a language: specific patterns combine into larger wholes.
- **How PSDC relates:** Rules and skills form a pattern language for research. Each rule (semantic logging, breath) addresses a recurring problem in autonomous collaboration. Together they form a coherent system.
- **Ref:** [A Pattern Language](https://global.oup.com/academic/product/a-pattern-language-9780195019193); [Archive](https://www.patternlanguage.com/)

## 5. Systems Design & Methodology Lineage

Approaches to structuring, designing, and reasoning about complex systems.

### Alexander: Design Patterns (1977) to Software Engineering
- **Core lineage:** Alexander's patterns were adopted by software engineering (Gang of Four, Design Patterns, 1994) and by agile methodologies. A pattern describes a problem + a context + a solution that recurs.
- **How PSDC relates:** Rules, skills, and hooks are design patterns. Each encodes a recurring problem (drift from research goals → breath rule; lost context → chronicle rule) and a solution pattern.
- **Ref:** [A Pattern Language](https://www.patternlanguage.com/); Gang of Four Design Patterns (1994)

### Meadows: System Dynamics & Leverage Points
- **Core lineage:** Understanding complex systems via stocks, flows, feedback loops, and interventions. High-leverage interventions reshape system behavior.
- **How PSDC relates:** Behavioral rules are high-leverage interventions. A single rule (e.g., semantic-logging) changes how the entire system operates without changing the underlying harness.
- **Ref:** [Thinking in Systems](https://research.fit.edu/media/site-specific/researchfitedu/coast-climate-adaptation-library/climate-communications/psychology-amp-behavior/Meadows-2008.-Thinking-in-Systems.pdf)

### Agile, Retrospectives, and Continuous Improvement
- **Core lineage:** Agile methodologies (Scrum, Kanban) use sprints + retrospectives to create feedback loops. The team reflects on what worked and adjusts.
- **How PSDC relates:** The breath rule (scheduled pause for trajectory check) and semantic logging (continuous reflection) formalize the agile retrospective pattern at the individual-agent level.
- **Ref:** Agile Manifesto (2001); various frameworks.

## Key Gaps: What Existing Work Doesn't Cover

**The verification-narrative integration:** Research automation systems (AI Scientist, GPT Researcher, STORM) excel at autonomy but provide no first-class verification mechanism. Memory systems (Mem0, Letta) manage persistence but assume learned compression. Philosophical frameworks (Extended Mind, Zettelkasten) describe why external structure matters but don't specify how to design it for *research verification*. PSDC uniquely combines: (1) persistent hand-authored state (like Zettelkasten philosophy), (2) explicit verification checkpoints (unlike autonomous pipelines), (3) behavioral rules as first-class citizens (like design patterns), and (4) human-AI symmetry in cognition (not automation-focused). No existing system treats the *narrative of verification* as a primary output alongside the research artifact itself. PSDC does.
